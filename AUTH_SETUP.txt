📊 Authentication Flow
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. User visits http://localhost:3000
   ↓
2. Clicks "Login" or "Sign Up"
   ↓
3. Enters credentials
   ↓
4. API validates and creates JWT token
   ↓
5. Token stored in httpOnly cookie
   ↓
6. Redirected to role-specific dashboard
   ↓
7. Middleware verifies token on protected routes
   ↓
8. User can access dashboard or logout

🔒 Security Implementation
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Password Storage:
  ✓ bcryptjs hashing with 10 salt rounds
  ✓ Passwords never stored in plain text
  ✓ Verified on login via bcryptjs.compare()

Session Management:
  ✓ JWT tokens with 7-day expiration
  ✓ Tokens signed with JWT_SECRET
  ✓ Verified on every protected request
  ✓ Stored in httpOnly cookies (not accessible via JS)
  ✓ Secure flag enabled in production
  ✓ SameSite=Lax to prevent CSRF

Authorization:
  ✓ Role-based access control (doctor vs patient)
  ✓ Middleware enforces route protection
  ✓ Automatic redirects based on role
  ✓ Token validation on /api/auth/me

Input Validation:
  ✓ Email format validation
  ✓ Password length (min 6 characters)
  ✓ Role validation (doctor or patient only)
  ✓ Email uniqueness on signup
  ✓ Required field checks

Login with Demo Account:
  1. Go to http://localhost:3000/auth/login
  2. Enter: dr.rajesh@clinsight.com
  3. Enter: password123
  4. Click Login
  5. Redirected to /doctor (doctor dashboard)

Create New Account:
  1. Go to http://localhost:3000/auth/signup
  2. Select: Patient
  3. Enter name, email, password
  4. Click "Create Account"
  5. Redirected to /auth/login
  6. Login with new account
  7. Redirected to /patient (patient portal)

Logout:
  1. Click logout icon (top right)
  2. Redirected to /auth/login

Protected Route Access:
  1. Try http://localhost:3000/doctor without auth
  2. Automatically redirected to /auth/login
  3. Login, then granted access to /doctor
